# Regras do Domínio Frontend (React / TypeScript / CSS)

Este diretório contém o frontend da aplicação Alethe. Ao trabalhar neste domínio, siga as diretrizes abaixo:

## Arquitetura Rápida

- `src/components/` — UI organizada por feature (ex: `HomeView/`, `WorkspaceView/`, `XTermView/`, `TitleBar/`, etc.). Cada componente possui seu próprio arquivo `.module.css`.
- `src/stores/` — Zustand: `projectsStore` (projetos, grupos, terminais, preferências, persistido em `projects.json`) e `uiStore` (modais, toasts, estados efêmeros).
- `src/lib/tauri.ts` — Wrapper de `invoke` para chamadas IPC ao backend Rust.
- `src/lib/i18n/` — Internacionalização. `messages/en.ts` é a fonte da verdade. O compilador valida se todas as chaves estão traduzidas em `pt-BR.ts`.
- `src/lib/types.ts` — Tipos de domínio reutilizáveis (`AgentType`, `Terminal`, `Project`, etc.).
- `src/styles/theme.css` — Variáveis de cores e tokens do design system.

## Convenções de Código

- Sempre crie **1 arquivo `.module.css` por componente**. Use tokens de estilo (`var(--bg)`, `var(--accent)`, etc.) e evite cores literais em CSS.
- Adicione novos tipos de domínio exclusivamente em `src/lib/types.ts`.
- Use seletores Zustand enxutos para evitar loops e re-renders desnecessários.
- Toda nova string visível deve ser envolvida em `t()` e cadastrada em `src/lib/i18n/messages/en.ts` e `src/lib/i18n/messages/pt-BR.ts`.
